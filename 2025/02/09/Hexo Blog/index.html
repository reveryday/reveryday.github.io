博客搭建：

（搭建指令皆在git bash中完成）

1、安装Git，安装好后检查版本信息：

$ git --version
git version 2.47.1.windows.2

2、安装nodejs（Hexo是基于nodejs编写的，需安装nodejs和里面的npm工具， npm会随Node.js一起安装），安装好后需将npm的全局包路径需要添加到系统的 PATH 环境变量中（通常为：C:\Users\用户名\AppData\Roaming\npm），再检查版本信息：

$ node -v
v22.13.0
$ npm -v
10.9.2

3、安装hexo，全局安装：

$ npm install -g hexo-cli

初始化：

$ hexo init Blog
$ npm install

up to date in 1s

24 packages are looking for funding
  run `npm fund` for details
$ ls
_config.landscape.yml  db.json    node_modules/      package.json  scaffolds/  themes/
_config.yml            deploy.sh  package-lock.json  public/       source/

4、生成SSH添加到GitHub

配置 Git 的全局用户名和邮箱：

$ git config --global user.name"reveryday"
$ git config --global user.email"wenswuu@gmail.com"

检查是否配置成功：

$ git config --global --list
user.name=reveryday
user.email=wenswuu@gmail.com
core.autocrlf=true

创建SSH密钥：

$ ssh-keygen -t rsa -C "wenswuu@gmail.com"

在GitHub的setting中，找到SSH keys的设置选项，点击New SSH key，把 C:\Users\34001.ssh\id_rsa.pub 里内容复制进去，查看是否成功：

$ ssh -T git@github.com
Hi reveryday! You've successfully authenticated, but GitHub does not provide shell access.

5、将hexo部署到GitHub

打开站点配置文件 _config.yml ，翻到最后，修改为（若default brunch是main，则应改为main）：

deploy:
  type: git
  repo: https://github.com/reveryday/reveryday.github.io.git
  branch: main

安装deploy-git （部署的命令）：

$ npm install hexo-deployer-git --save

安装完成后：

$ hexo clean
$ hexo g
$ hexo d

4、other issues：

1、bash: hexo: command not found（突然无法找到hexo指令）

$ hexo clean
bash: hexo: command not found

尝试了重新全局安装hexo也没用：

$ npm install -g hexo-cli

changed 53 packages in 7s

14 packages are looking for funding
  run `npm fund` for details

$ hexo clean
bash: hexo: command not found

gpt给出的原因是环境变量未更新：

在安装全局 npm 包时，npm 的全局包路径需要添加到系统的 PATH 环境变量中，如果环境变量未更新，系统无法找到全局安装的包，只需打开系统环境变量设置界面，在系统变量中找到 Path，点击编辑，添加 npm 全局包路径，通常为：

C:\Users\用户名\AppData\Roaming\npm

添加完成后重启终端即可

2、html语法部署到hexo中图片无法显示

插入图片方式：<img src="images/屏蔽设计02.png" style="width:80%;">，对应网页源代码：

<h3 id="Build-Up-Factor"><a href="#Build-Up-Factor" class="headerlink" title="Build Up Factor"></a>Build Up Factor</h3><img src="images/屏蔽设计02.png" style="width:80%;">

使用markdown语法：![](images/屏蔽设计02.png)插入图片对应源代码：

<h3 id="Build-Up-Factor"><a href="#Build-Up-Factor" class="headerlink" title="Build Up Factor"></a>Build Up Factor</h3><p><img src="/images/%E5%B1%8F%E8%94%BD%E8%AE%BE%E8%AE%A102.png"></p>

当在 .md 文件中为![](images/屏蔽设计02.png)，Hexo 会自动解析图片路径，并结合资源管理机制（例如 post_asset_folder 或全局 source/images/)生成正确的静态路径，在生成静态文件时，Hexo 将 images/屏蔽设计02.png 转换为绝对路径 /images/屏蔽设计02.png，并确保图片被复制到 public/images/，部署后，网页源代码中的 <img src="/images/%E5%B1%8F%E8%94%BD%E8%AE%BE%E8%AE%A102.png"> 显示为 URL 编码形式（屏蔽设计02 被编码为 %E5%B1%8F%E8%94%BD%E8%AE%BE%E8%AE%A102），浏览器会解码并请求 /images/屏蔽设计02.png，如果文件存在，就能显示。

而当直接写 <img src="images/屏蔽设计02.png" style="width:80%;">，Hexo 的 Markdown 渲染器不会对 HTML 中的图片路径进行处理，部署后，这个路径会原样写入生成的 HTML 文件，这里 images/屏蔽设计02.png 是相对于当前页面（例如 /build-up-factor/）的相对路径，而不是网站根目录的绝对路径（/images/屏蔽设计02.png），实际请求的 URL 可能是 /build-up-factor/images/屏蔽设计02.png，但 public/build-up-factor/images/ 下并没有这个文件，导致 404 错误。

Hexo 的 post_asset_folder 或全局资源文件夹只对 Markdown 中的图片生效，HTML 中的 路径需要手动确保正确，如果图片在 source/images/ 下，HTML 应该用绝对路径 <img src="/images/屏蔽设计02.png">，而不是相对路径 images/屏蔽设计02.png

  • Markdown语法的成功：Hexo 和 Butterfly 主题自动将 Markdown 中的相对路径 images/屏蔽设计02.png 转换为正确的绝对路径 /images/屏蔽设计02.png，并确保图片在 public/images/ 中。
  • HTML 语法的失败：<img src="images/屏蔽设计02.png"> 被视为相对路径，部署后解析为错误的 URL（例如 /build-up-factor/images/屏蔽设计02.png），而正确路径应为 /images/屏蔽设计02.png。

4、latex公式在typora与部署到hexo上渲染冲突的问题

在typora上可以正常显示而hexo不能：

$$ 
y_{i} = \left\{ \begin{array}{ll} 1, & \text{if } i = y \\ 0, & \text{otherwise} \end{array} \right.   
$$

在hexo正常而typora上无法渲染成功：

<div style="text-align: center;">
  $$ y_{i} = \left\{ \begin{array}{ll} 1, & \text{if } i = y \\ 0, & \text{otherwise} \end{array} \right. $$
  <p style="margin: 0; text-align: center;">
</div>

解决方法：公式中去掉html标签，配置 Hexo 支持 LaTeX，Hexo 需要额外的插件来渲染 LaTeX

npm uninstall hexo-renderer-marked
npm install hexo-renderer-kramed
npm install hexo-math
