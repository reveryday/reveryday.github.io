Fotran environment configuration

01、vscode+fortran

VSCode 并不强制使用 Intel 编译器，可以直接用 gfortran编译 Fortran 代码，gfortran 是 GNU Fortran 编译器，开源免费，适用于普通 Fortran 开发。

issue 1: The Fortran Language Server server crashed 5 times in the last 3 minutes. The server will not be restarted.

vscode也可以直接配置已经安装了的oneapi里的Fortran编译器、

02、vs2022+fortran

需安装VS2022与oneAPI，VS2022 主要用于 Intel Fortran 编译器（ifort / ifx），不支持 GNU Fortran（gfortran），必须安装 Intel OneAPI，才能让 VS2022 识别 Fortran 代码并编译，Intel OneAPI 里面包含 Intel Fortran 编译器（ifort / ifx）。

VS2022安装时只需勾选：

oneAPI安装：访问官网（网址：https://www.intel.cn/content/www/cn/zh/developer/tools/oneapi/toolkits.html ），先安装基础工具包 Intel oneAPI Base Toolkit，再安装 Intel oneAPI HPC Toolkit，安装Intel oneAPI Base Toolkit时，如果全部安装所占内存太大，所以我只勾选了两个包：

安装Intel oneAPI HPC Toolkit 我只勾选了一个包：

在VS2022上测试时发现无法新建一个 Fortran 项目（没有 Fortran 语言选项），可能是Intel oneAPI HPC Toolkit安装时没有勾选Intel Fortran Compiler，VS安装这两个插件就解决了：

建立Fortran项目测试：新建一个 Fortran 语言的 Main Program Code 项目，会出现内置的 Fortran 示例代码 （.f90文件），运行测试即可，贴一下可用于测试的示例代码：

    implicit none 
    print *, 'Hello World'
    pause 
end program Console1

Others

01、安装MiKTeX时，

打开相关日志（C:\Users\34001\AppData\Local\MiKTeX\miktex\log\initexmf.log）如下：

找到了原因，是因为我在用户变量与系统变量里有着一条：

D:\LANL\MCNP6\MCNP_CODE\MCNP6\bin\mcnp6.exe

把用户变量与系统变量里对应的mcnp6.exe去掉，改成D:\LANL\MCNP6\MCNP_CODE\MCNP6\bin就行了。MiKTeX 的初始化工具（比如 initexmf）会扫描系统环境变量（如 PATH），以确定某些外部程序或依赖项的位置，如果 PATH 中包含了错误路径（D:\LANL\MCNP6\MCNP_CODE\MCNP6\bin\mcnp6.exe\），MiKTeX 可能会误以为这是一个有效的目录，而不是一个文件，导致尝试访问它时出错。
